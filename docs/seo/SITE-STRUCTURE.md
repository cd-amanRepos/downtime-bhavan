### URL conventions

- All URLs lowercase, hyphenated
- No trailing slash (Next.js default)
- No `.html` extension
- No tracking params in canonical (`utm_*` stripped or canonical points to clean URL)
- Site IDs use kebab-case based on portal name: `aadhaar-uidai`, `epfo`, `gst-portal`, `income-tax-india`, `pan-nsdl`, `passport-seva`, `digilocker`, `eshram`, `esic`, `mca21`, `irctc`, `fastag-ihmcl`
- Dates in `/daily/` use 4-digit year + 2-digit month
- Daily Downtime slugs: 4–7 words, dash-separated, no stop words: `uidai-down-47-minutes-march-23`

## Sitemap quality gates

Every URL listed in `sitemap.xml` MUST:

- [ ] Return HTTP 200 (no 3xx, no 4xx, no 5xx)
- [ ] Be indexable (no `noindex` meta, no `X-Robots-Tag: noindex`)
- [ ] Have unique `<title>` (30–60 chars)
- [ ] Have unique `<meta name="description">` (120–160 chars)
- [ ] Have a self-referencing canonical
- [ ] Have at least 300 words of unique content OR be a Dataset/Tool page with structured data substitute
- [ ] Have at least one incoming internal link from another sitemap URL
- [ ] Have valid OpenGraph + Twitter Card meta

URLs explicitly **excluded** from `sitemap.xml`:

- `/admin/*` — gated, also blocked in robots.txt
- `/api/*` except `/api/public/uptime.json`
- `/delete-my-data` confirmation flow (process page, not content)
- Old / redirected URLs (404 or 301 only)
- Search result pages (none exist; if added later, exclude)
- Pagination beyond page 1 of `/daily` (canonical to page 1)

| `/methodology` | nav back | examples link to actual sites | "See live data" | "How crowd reports are validated" | — | yes | — | reports citing methodology |
| `/about` | nav back | — | — | — | yes | — | — | — |
| `/daily/[slug]` | nav back | always link to the site(s) discussed | yes if uptime mentioned | yes if grievance volume mentioned | yes if methodology mentioned | author byline | "Related posts" (3) | yes if report-relevant |
| `/reports/[slug]` | nav back | yes (data per site) | yes (rankings cited) | yes | yes (methodology footnote) | author | — | "Other reports" |
| `/press` | nav back | — | yes (data sample) | — | yes | yes | "Recent coverage" | "Read our reports" |
| `/donate` | nav back | — | — | — | — | yes | — | — |

**Linking principles:**

1. Every tier-1 page (home, sites, leaderboard, methodology) is reachable in ≤2 clicks from any other page.
2. No page is orphaned (sitemap quality gate).
3. Anchor text varies — never always "click here"; use descriptive phrases ("see how we check UIDAI", "30-day uptime leaderboard").
4. No internal `nofollow` (no PageRank sculpting).
5. Footer links: home, sitemap, methodology, about, press, donate, privacy, contact, delete-my-data, llms.txt. Limit ~10 to avoid footer-spam signal.

## Canonical map

| URL pattern | Canonical | Notes |
|---|---|---|
| `/?utm_source=x` | `/` | All UTM params stripped |
| `/sites/aadhaar-uidai` | `/sites/aadhaar-uidai` | Self-canonical |
| `/sites/aadhaar-uidai?ref=share` | `/sites/aadhaar-uidai` | Share params stripped |
| `/daily/2026/07/uidai-down-47-minutes` | self | Self-canonical |
| `/daily/2026/07/uidai-down-47-minutes/` | `/daily/2026/07/uidai-down-47-minutes` | Trailing slash → no trailing |
| `/leaderboard?sort=worst` | `/leaderboard` | Filter params don't get separate canonical (filters are client-side state) |
| `/janta-darbar?site=epfo` | `/janta-darbar` | Filter params don't get separate canonical |
| Old/redirected URLs | redirect target | Use 301 |

**Implementation:** Use `metadata.alternates.canonical` per route in Next.js. Combine with `next.config.ts` redirects for hard redirects.

## Per-page metadata spec

Every page needs:

```ts
export const metadata: Metadata = {
  title: '...',                            // 30–60 chars, unique
  description: '...',                      // 120–160 chars, unique, includes primary keyword
  alternates: { canonical: '/path' },      // self-canonical
  openGraph: {
    title: '...',                          // can match title; often re-shaped for social
    description: '...',                    // can match description; often punchier for social
    url: 'https://downtimebhavan.in/path',
    siteName: 'Downtime Bhavan',
    locale: 'en_IN',
    type: 'website' | 'article',           // article for /daily, /reports
    images: [{ url: '...', width: 1200, height: 630, alt: '...' }],
  },
  twitter: {
    card: 'summary_large_image',
    title: '...',
    description: '...',
    images: ['...'],
  },
  robots: {
    index: true,
    follow: true,
    googleBot: {
      index: true,
      follow: true,
      'max-image-preview': 'large',
      'max-snippet': -1,
      'max-video-preview': -1,
    },
  },
};
```

## Hreflang policy

Single locale: `en-IN`. Hindi micro-touches are not a separate locale and do **not** warrant a separate URL or hreflang entry. Setting:

```html
<link rel="alternate" hreflang="en-IN" href="https://downtimebhavan.in/[path]" />
<link rel="alternate" hreflang="x-default" href="https://downtimebhavan.in/[path]" />
```

Revisit only if a full Hindi-language route set is launched in year 2+.

## Pagination

- `/daily` archive paginates at 20 posts per page
- Use `rel="canonical"` pointing to page 1 from page 2+
- Use distinct titles ("The Daily Downtime · page 2") — do not noindex paginated pages
- `/reports` will not paginate until 20+ reports exist
